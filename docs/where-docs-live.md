# Where documentation lives

The full, always-current map lives in the docs portal: **https://docs.npp.whiteforge.ai** → Contributing → "Where documentation lives".

Quick reference:

| What | Where |
| --- | --- |
| Service build/run/test, internals | the service repo's `README.md` + `AGENTS.md` |
| Platform architecture, ADRs, IAM how-tos | du-docs (`docs.npp.whiteforge.ai`) |
| Product / end-user docs (Ukrainian) | du-docs product sections |
| Engineering process (branching, commits, PRs, versioning, releases, observability, repo structure) | this repo, `docs/` |
| API contracts / OpenAPI | `du-api-contracts` repo |

Technical docs are English; product/user docs are Ukrainian. One canonical home per topic — link, don't copy. `digital-university-architecture` is obsolete.
