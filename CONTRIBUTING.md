## About these conventions

These conventions exist so that every contributor works in a predictable
environment. When branches, commits, pull requests, versions, and releases
follow the same rules across all repositories, it becomes easier to onboard
new people, review each other's work, trace changes, and ship with confidence.

Most of these rules are enforced automatically through GitHub Rulesets and CI.
The rest are conventions that we follow as a team.
